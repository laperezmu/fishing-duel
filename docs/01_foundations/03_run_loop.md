# Loop principal de run

## Estado

- Estado: `pendiente`
- Prioridad: `critica`
- Depende de: `01_game_vision`, `02_design_pillars`

## Objetivo

Definir el equivalente a `blind -> shop -> siguiente blind` para este juego.

## Preguntas a resolver

- cual es la secuencia exacta de una run
- en que momentos aparece la tienda
- cuando se valida la cuota de zona
- como se entra a la zona legendaria

## Estructura esperada

1. Inicio de run
2. Eleccion de lance
3. Encuentro
4. Resultado del encuentro
5. Progreso de cuota o avance
6. Tienda o transicion
7. Siguiente decision

## Criterio de cierre

- La run puede describirse en 6 a 10 pasos maximo.
- Cada paso indica que decide el jugador y que recurso cambia.
