# Docs

Esta carpeta combina una guia operativa activa del workflow actual con documentacion legacy que se conserva solo como referencia.

## Estado

- `docs/workflow-github-issues-y-specify.md`: guia operativa activa del backlog y delivery.
- `docs/features/`: planes legacy ya cerrados o archivados.
- `docs/discoveries/`: discoveries historicos conservados solo como referencia.

## Regla actual

- No crear nuevas features activas ni nuevos backlogs locales en `docs/`.
- El flujo oficial vive en GitHub Issues, `specs/` y `.specify/`.
- Si una feature nueva necesita contexto legacy, debe enlazar explicitamente los docs relevantes desde su spec activa.

## Referencias activas

- `.specify/memory/constitution.md`: principios y gobierno del nuevo framework.
- `docs/workflow-github-issues-y-specify.md`: flujo operativo del repo.
- `.specify/templates/overrides/`: plantillas adaptadas al proyecto.
- `specs/`: specs, planes y tareas activas.
