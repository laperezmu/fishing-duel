# Docs

La documentacion del proyecto se divide en tres capas principales.

## Estructura

- `docs/features/`: planes de features trabajadas bajo el flujo de implementacion.
- `docs/discoveries/`: documentos activos de discovery y resultados de investigacion.
- `docs/backlog/`: backlog de investigacion, priorizacion y desarrollo futuro del juego.

## Regla practica

- Si una tarea ya se va a implementar, se crea como plan en `docs/features/`.
- Si una idea aun necesita discovery, definicion o priorizacion, vive en `docs/backlog/`.
- Si el discovery ya esta en marcha o produjo un documento de referencia, vive en `docs/discoveries/`.

## Referencias

- La plantilla de plan de feature sigue definida en `PROJECT_CONTEXT.md`.
- Los discoveries activos se documentan en `docs/discoveries/`.
- El backlog tiene su propia guia y template en `docs/backlog/`.
