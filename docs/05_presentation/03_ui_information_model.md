# Modelo de informacion visible

## Estado

- Estado: `pendiente`
- Prioridad: `alta`

## Objetivo

Definir que debe saber el jugador en cada pantalla para tomar decisiones con claridad.

## Debe incluir

- informacion visible en seleccion de lance
- informacion visible en combate
- informacion visible en tienda
- informacion visible en avance de zona y fin de run

## Regla Balatro

La informacion importante debe estar siempre disponible y ser facil de leer.

## Criterio de cierre

- Cada pantalla clave tiene lista de informacion obligatoria.
- La claridad de riesgo y recompensa esta cubierta.
