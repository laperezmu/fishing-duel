# Catalogo de zonas

## Estado

- Estado: `pendiente`
- Prioridad: `critica`
- Depende de: `01_foundations/03_run_loop.md`, `01_foundations/04_fail_success_conditions.md`

## Objetivo

Definir las zonas del MVP, su identidad y su funcion dentro del arco de dificultad.

## Cada zona debe especificar

- fantasia o bioma
- cuota de avance
- especies principales
- tipo de riesgo dominante
- recompensas o promesa de build
- regla especial si aplica

## Criterio de cierre

- Todas las zonas del MVP estan listadas.
- Cada zona tiene identidad distinta.
- La progresion entre zonas se entiende sin apoyo externo.
