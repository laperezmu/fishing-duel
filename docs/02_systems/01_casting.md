# Sistema de lances

## Estado

- Estado: `pendiente`
- Prioridad: `critica`
- Depende de: `01_foundations/03_run_loop.md`

## Objetivo

Definir que ofrece cada tipo de lance y por que el jugador elegira uno sobre otro.

## Temas a cerrar

- corto, medio y largo
- que cambia en tiempo de espera
- que cambia en riesgo inicial
- que cambia en pool de peces
- que cambia en reputacion

## Regla Balatro

Cada lance debe ser una opcion con nicho propio. Ninguno debe ser correcto siempre.

## Criterio de cierre

- Cada lance tiene identidad.
- Cada lance tiene una recompensa y un costo claros.
- Se entiende cuando conviene usar cada uno.
